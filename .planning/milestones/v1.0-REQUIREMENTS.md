## v1 Requirements (Complete)

### Model Infrastructure

- [x] **MODEL-01**: App downloads LLM model (~1.8GB) with visible progress indicator
- [x] **MODEL-02**: Download can be paused and resumed if interrupted
- [x] **MODEL-03**: App initializes model and handles initialization errors gracefully
- [x] **MODEL-04**: App monitors memory pressure and degrades gracefully when iOS needs RAM
- [x] **MODEL-05**: Model state persists across app launches (no re-download needed)

### Chat Experience

- [x] **CHAT-01**: User can send text messages and receive AI responses
- [x] **CHAT-02**: AI responses stream token-by-token as they generate
- [x] **CHAT-03**: Typing indicator shows while model generates response
- [x] **CHAT-04**: User can tap microphone to speak instead of type (speech-to-text)
- [x] **CHAT-05**: Speech recognition runs on-device (no cloud transcription)
- [x] **CHAT-06**: Messages persist to local storage and survive app restart
- [x] **CHAT-07**: User can view conversation history
- [x] **CHAT-08**: App detects when conversation ends (app background or explicit end)
- [x] **CHAT-09**: System prompt creates warm, empathetic AI personality

### Memory System

- [x] **MEM-01**: After conversation ends, app extracts key information (people, events, emotions, facts)
- [x] **MEM-02**: Extracted memories store with importance scores
- [x] **MEM-03**: Memories have decay rates (persistent facts decay slowly, ephemeral events decay quickly)
- [x] **MEM-04**: When building prompts, app retrieves relevant memories based on recency and importance
- [x] **MEM-05**: System prompt includes relevant memories for conversation continuity
- [x] **MEM-06**: Context window management keeps prompts under 4096 tokens

### Safety

- [x] **SAFE-01**: App scans user messages for crisis language before sending to model
- [x] **SAFE-02**: Crisis detection triggers modal with hotline numbers (988, Crisis Text Line)
- [x] **SAFE-03**: Crisis modal is non-dismissable for 5 seconds
- [x] **SAFE-04**: User can continue conversation after acknowledging crisis resources
- [x] **SAFE-05**: App includes disclaimer that it's not therapy (shown in onboarding)
- [x] **SAFE-06**: Disclaimer accessible from settings screen

### Polish

- [x] **POLISH-01**: First launch shows onboarding flow explaining privacy and local-only approach
- [x] **POLISH-02**: Onboarding collects user's name for personalized responses
- [x] **POLISH-03**: Onboarding displays and requires acknowledgment of disclaimer
- [x] **POLISH-04**: Settings screen provides access to crisis resources
- [x] **POLISH-05**: Settings screen allows user to clear all data
- [x] **POLISH-06**: App has custom icon and splash screen
- [x] **POLISH-07**: App ready for TestFlight distribution

## Out of Scope (v1)

Deferred features documented to prevent scope creep:

| Feature | Reason |
|---------|--------|
| Cloud sync | Destroys privacy value proposition; conflicts with local-first architecture |
| User accounts | Unnecessary for local-only app; adds complexity |
| Social sharing | Conflicts with cove positioning; privacy concerns |
| Android support | iOS-first strategy; may revisit after v1 validation |
| Gamification (streaks, badges) | Mental health isn't a game; can create anxiety |
| Professional therapist integration | Scope creep; regulatory complexity; liability |
| Romantic AI features | Ethical concerns; reputation risk |
| Real-time voice conversation | High complexity; focus on voice input with text output for v1 |
| HealthKit integration | Scope creep; defer to v2+ |
| Multiple conversation threads | Added complexity; single thread sufficient for v1 |
