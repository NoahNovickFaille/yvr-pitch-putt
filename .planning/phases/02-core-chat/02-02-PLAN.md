Create the crisis detection function that scans user messages.

Interface:
```typescript
export interface CrisisResult {
  detected: boolean;
  matchedPhrases: string[];
  severity: 'low' | 'medium' | 'high';
}

export function detectCrisis(message: string): CrisisResult;
```

Logic:
1. Normalize message: lowercase, trim whitespace
2. Check for negation patterns BEFORE phrase matching
   - If message contains negation before a crisis phrase, reduce severity
   - Example: "I don't want to die" should NOT trigger high severity
   - Simple heuristic: if negation word appears within 10 characters before phrase
3. Scan for HIGH_SEVERITY_PHRASES - any match = high severity (unless negated)
4. Scan for MEDIUM_SEVERITY_PHRASES - 2+ matches = medium severity
5. Return result with matched phrases for logging/debugging

Edge cases:
- Empty message: return {detected: false, matchedPhrases: [], severity: 'low'}
- Very short message (<3 chars): same as empty
- Negated phrase: demote from high to medium, or medium to low

Add console.log for detected crises (for debugging during development - can be removed later).
