  <name>Task 1: Create memory type definitions</name>

  <action>
Create memory type definitions following the pattern from chat.ts:

1. Define MemoryType as union: 'person' | 'event' | 'emotion' | 'fact' | 'preference'
   - person: Names, relationships mentioned (e.g., "Sarah is my sister")
   - event: Things that happened or will happen
   - emotion: Feelings expressed
   - fact: Factual information about user
   - preference: Likes, dislikes, preferences

2. Define MemoryCategory as union: 'persistent' | 'temporal' | 'contextual'
   - persistent: Facts unlikely to change (slow decay - 168 hour half-life)
   - temporal: Time-bound events/emotions (medium decay - 24 hour half-life)
   - contextual: Conversation-specific details (fast decay - 4 hour half-life)

3. Define Memory interface with:
   - id: string
   - type: MemoryType
   - content: string (the extracted information)
   - importance: number (1-10 scale)
   - category: MemoryCategory
   - createdAt: number (timestamp)
   - lastAccessed: number (timestamp, for decay reinforcement)
   - accessCount: number (retrieval count)
   - sourceConversationId?: string (optional, for debugging)

4. Define ExtractionResult interface for LLM output:
   ```typescript
   interface ExtractionResult {
     memories: Array<{
       type: MemoryType;
       content: string;
       importance: number;
       category: MemoryCategory;
     }>;
   }
   ```

5. Create generateMemoryId helper following chat.ts pattern:
   `mem-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`

Create the extraction prompt and JSON schema for structured LLM output:

1. Create directory: src/services/memory/

2. Define MEMORY_EXTRACTION_PROMPT as a concise instruction string:
   - Instruct LLM to analyze conversation and extract user information
   - Extract ONLY information from USER messages (not assistant)
   - Explain the 5 memory types with examples
   - Explain importance scoring (1-10):
     * 10: Core identity (name, major relationships, profession)
     * 7-9: Significant life events, important people
     * 4-6: Notable preferences, recurring topics
     * 1-3: Minor details, passing mentions
   - Explain category assignment (affects decay rate)
   - Instruct: "Output JSON only. Extract 0-8 memories per conversation."

3. Define MEMORY_EXTRACTION_SCHEMA as JSON Schema object:
   ```typescript
   export const MEMORY_EXTRACTION_SCHEMA = {
     type: 'object',
     properties: {
       memories: {
         type: 'array',
         items: {
           type: 'object',
           properties: {
             type: { type: 'string', enum: ['person', 'event', 'emotion', 'fact', 'preference'] },
             content: { type: 'string', maxLength: 200 },
             importance: { type: 'number', minimum: 1, maximum: 10 },
             category: { type: 'string', enum: ['persistent', 'temporal', 'contextual'] }
           },
           required: ['type', 'content', 'importance', 'category']
         },
         maxItems: 8
       }
     },
     required: ['memories']
   };
   ```

Keep the prompt concise - it will be prepended to conversation text, consuming tokens.
