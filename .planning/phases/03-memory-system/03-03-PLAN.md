  <action>
Create the memory extraction service:

1. Import dependencies:
   ```typescript
   import { LLMService } from '../llm/LLMService';
   import { ChatMessage } from '../../types/chat';
   import { ExtractionResult, MemoryType, MemoryCategory } from '../../types/memory';
   import { MEMORY_EXTRACTION_PROMPT, MEMORY_EXTRACTION_SCHEMA } from './extractionPrompt';
   ```

2. Create formatConversationForExtraction function:
   - Takes: messages (ChatMessage[]), maxMessages (number = 20)
   - Slices to last N messages to limit token usage
   - Formats as "User: {content}\nAssistant: {content}\n..."
   - Returns formatted string
   - Export this function (useful for testing)

3. Create extractMemories async function:
   - Takes: conversationText (string)
   - Returns: Promise<ExtractionResult['memories']>

   Implementation steps:
   a. Check if LLM is ready, return empty array if not
   b. Get context: `LLMService.getContext()`
   c. Call completion with json_schema:
      ```typescript
      const result = await context.completion({
        messages: [
          { role: 'system', content: MEMORY_EXTRACTION_PROMPT },
          { role: 'user', content: conversationText }
        ],
        response_format: {
          type: 'json_schema',
          json_schema: { schema: MEMORY_EXTRACTION_SCHEMA }
        },
        n_predict: 512,        // Enough for ~8 memories
        temperature: 0.3,      // Lower for consistent extraction
        top_p: 0.9,
      });
      ```
   d. Parse result with try-catch:
      ```typescript
      try {
        const parsed: ExtractionResult = JSON.parse(result.text);
        return parsed.memories || [];
      } catch (error) {
        console.error('[MemoryExtractor] JSON parse failed:', error);
        console.error('[MemoryExtractor] Raw output:', result.text);
        // Retry once with explicit JSON instruction
        return await retryExtraction(conversationText, context);
      }
      ```

4. Create retryExtraction helper (private):
   - Add explicit instruction: "Output valid JSON only. Your previous response was not valid JSON."
   - Same completion parameters
   - If parse fails again, return empty array (log error)

5. Add DEV logging for:
   - Extraction start (message count, text length)
   - Extraction complete (memory count)
   - Parse failures

Note: This extracts memories but does NOT store them. Storage happens in the orchestration layer (next plan).
