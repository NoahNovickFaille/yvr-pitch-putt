Modify root layout to check onboarding status and route appropriately.

**Current structure (from existing app/_layout.tsx):**
- Stack routing with (tabs) anchor
- Store initialization in useEffect (loadFromStorage)
- ThemeProvider wrapping Stack with nested Stack.Screen components

**app/_layout.tsx modifications:**

1. Import SplashScreen and call preventAutoHideAsync() in GLOBAL SCOPE (before RootLayout function):
```typescript
import * as SplashScreen from 'expo-splash-screen';
SplashScreen.preventAutoHideAsync();
```

2. Import useOnboardingStore and OnboardingScreen:
```typescript
import { useOnboardingStore } from '@/src/stores/onboardingStore';
import { OnboardingScreen } from '@/src/screens/OnboardingScreen';
```

3. Add state: isReady (boolean, default false) using useState

4. Modify existing useEffect to include onboarding check:
```typescript
useEffect(() => {
  console.log('[RootLayout] Initializing stores from storage');
  useChatStore.getState().loadFromStorage();
  useMemoryStore.getState().loadFromStorage();

  // Check onboarding status
  useOnboardingStore.getState().checkOnboardingStatus();

  // Mark ready (synchronous operations only)
  setIsReady(true);
}, []);
```

5. Add separate useEffect watching isReady:
```typescript
useEffect(() => {
  if (isReady) {
    SplashScreen.hideAsync();
  }
}, [isReady]);
```

6. Get onboarding state:
```typescript
const onboardingCompleted = useOnboardingStore((state) => state.completed);
```

7. **Conditional rendering BEFORE ThemeProvider:**
```typescript
// Show nothing while initializing (splash remains visible)
if (!isReady) {
  return null;
}

// Show onboarding if not completed
if (!onboardingCompleted) {
  return <OnboardingScreen />;
}

// Otherwise render normal app (existing ThemeProvider + Stack)
return (
  <ThemeProvider value={colorScheme === 'dark' ? DarkTheme : DefaultTheme}>
    ...existing Stack structure...
  </ThemeProvider>
);
```

**Integration approach:** Conditional rendering at root level, NOT nested in Stack routing. This keeps onboarding separate from navigation structure.

**Important:** Do NOT move existing model download logic or store initialization. Only ADD onboarding check and conditional rendering.

Follow research example from 04-RESEARCH.md Pattern 2 "Manual Splash Screen Control".
