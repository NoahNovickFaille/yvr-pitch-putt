**Build steps:**

```bash
grep "bundleIdentifier" app.config.js
```

2. Install EAS CLI if needed:
```bash
which eas || npm install -g eas-cli
```

3. Login to Expo:
```bash
eas login
```

4. Configure project:
```bash
eas build:configure
```

5. Trigger production build:
```bash
eas build --platform ios --profile production
```

This will:
- Upload source code to EAS servers
- Build IPA with managed credentials
- Provide build URL for monitoring

Build takes 10-20 minutes. Do NOT wait for completion - this is checkpoint gate.

## Required: Apple Developer Account Setup

**Before proceeding, you need:**

1. **Apple Developer Program membership** ($99/year)
   - Enroll at: https://developer.apple.com/programs/
   - Approval takes 24-48 hours if not already enrolled

2. **App Store Connect app record**
   - Visit: https://appstoreconnect.apple.com/
   - Click "+" → "New App"
   - Bundle ID: ca.corvustech.confidant
   - Name: Confidant
   - Primary Language: English
   - Note the App ID (numeric, e.g., 1234567890)

3. **Get your credentials:**
   - Apple ID: Your developer account email
   - ASC App ID: From step 2 (numeric ID)
   - Team ID: In App Store Connect → Membership

## TestFlight Submission Steps

**Option A: Simple command (recommended)**
```bash
npx testflight --what-to-test "Initial beta: onboarding, chat, memory system, crisis detection complete"
```

This single command handles:
- Waiting for EAS build to complete
- Downloading IPA
- Uploading to App Store Connect
- Submitting for Beta App Review

**Option B: Manual EAS submit**
```bash
# Update eas.json with your credentials:
# - appleId: your developer email
# - ascAppId: numeric app ID from App Store Connect
# - appleTeamId: from membership page

eas submit --platform ios --profile production
```

**Wait for Beta App Review:**
- Apple reviews TestFlight builds separately from App Store Review
- Typical timeline: 1-2 days
- You'll receive email when approved

## Verification

**Internal Testing (immediate):**
1. Open TestFlight app on iOS device
2. Build appears automatically (you're internal tester as developer)
3. Tap Install
4. Launch app from TestFlight

**Test checklist:**
- [ ] App icon displays correctly on home screen
- [ ] Splash screen shows briefly on launch
- [ ] First launch shows onboarding (Privacy → Name → Disclaimer)
- [ ] Complete onboarding → chat/setup screen appears
- [ ] Settings tab shows crisis resources and disclaimer
- [ ] Clear All Data works and doesn't reset onboarding

**External Testing (after Beta App Review):**
1. In App Store Connect → TestFlight → External Groups
2. Create group, add testers by email
3. Once Apple approves, testers receive email invite
4. Testers install via TestFlight app

## Troubleshooting

**Build fails:**
- Check EAS build logs: `eas build:list` then click URL
- Common issue: Certificate/provisioning profile (EAS handles automatically, but verify Apple Developer account is active)

**Submit fails - authentication:**
- Run: `eas credentials` to verify Apple account connection
- May need to generate App Store Connect API key

**Beta App Review rejection:**
- Check email for specific reasons
- Usually needs disclaimer to be more prominent (already in onboarding)
- Resubmit with clarifications
