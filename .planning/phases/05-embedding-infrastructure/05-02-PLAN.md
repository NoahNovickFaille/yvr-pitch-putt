Create `src/services/embedding/EmbeddingService.ts` following the LLMService pattern:

1. Import dependencies:
   - initLlama, LlamaContext from 'llama.rn'
   - getInfoAsync from 'expo-file-system/legacy'
   - EMBEDDING_MODEL, getEmbeddingModelPath from constants/embedding
   - EmbeddingServiceState, EmbeddingServiceStatus, EmbeddingVector from types/embedding

2. Create `EmbeddingServiceImpl` class:
   - Private static instance for singleton
   - Private state: EmbeddingServiceState (starts as { status: 'idle' })
   - Private listeners: Set<StateListener> for reactivity
   - Private initPromise: Promise<void> | null for deduplication
   - Private context: LlamaContext | null

3. Implement public methods:
   - `getInstance()`: Return singleton
   - `subscribe(listener)`: Subscribe to state changes, return unsubscribe function
   - `getState()`: Return current state
   - `isReady()`: Return status === 'ready' && context !== undefined
   - `isModelDownloaded()`: Check if embedding model file exists (async)

4. Implement `initialize()`:
   - Guard: If already ready, return immediately
   - Guard: If initPromise exists, return it (prevent concurrent init)
   - Set initPromise = doInitialize()
   - Return initPromise

5. Implement `doInitialize()` (private):
   - Set state to 'initializing'
   - Get model path via getEmbeddingModelPath()
   - Check file exists with getInfoAsync
   - If not exists, throw 'Embedding model not downloaded'
   - Call initLlama with EMBEDDING_MODEL.llm config (MUST include embedding: true)
   - Set state to 'ready' with context
   - Clear initPromise
   - On error: set state to 'error' with message, clear initPromise, throw

6. Implement `embed(text: string)`: Promise<EmbeddingVector>
   - Throw if not ready
   - Call this.context!.embedding(text)
   - Return result.embedding (Array<number>)
   - Log timing in __DEV__ mode

7. Implement `release()`:
   - If ready, set state to 'unloaded', clear context reference
   - Called on memory warning or when not needed

8. Export singleton: `export const EmbeddingService = EmbeddingServiceImpl.getInstance()`

Key differences from LLMService:
- Uses embedding: true in llama config
- Uses EMBEDDING_MODEL constants (not MODEL_CONFIG)
- No completion queue (just embed() method)
- Separate model file (all-MiniLM vs Llama)

# Embedding Service

On-device semantic similarity using all-MiniLM-L6-v2 via llama.rn.

## Key Files

- **EmbeddingService.ts** - Singleton managing llama.rn context for embeddings
- Uses separate context from chat LLM (embedding: true required)
- Model: all-MiniLM-L6-v2-Q4_K_M.gguf (21MB, 256-dim output)

## Usage

```typescript
// Initialize (call after model is downloaded)
await EmbeddingService.initialize();

// Generate embedding
const vector = await EmbeddingService.embed("user likes hiking");
// vector is number[] with 256 elements

// Release on memory pressure
EmbeddingService.release();
```

## Important Notes

- NEVER share context with LLMService - embedding models require separate init
- Initialize lazily (only when needed for memory extraction)
- Release after batch operations to conserve memory
- Model must be downloaded before initialize() will succeed
