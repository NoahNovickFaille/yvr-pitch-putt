<objective>
Create migration system for embedding existing memories that were created before the embedding infrastructure.

Purpose: Users upgrading from v1.0 have memories without embeddings. This plan creates a background migration task that embeds existing memories, enabling semantic deduplication for all memories (not just new ones).

Output: EmbeddingMigration service that runs in background on first launch after upgrade, useEmbeddingMigration hook for progress tracking.
</objective>

   - EmbeddingService from ./EmbeddingService

   - { useMemoryStore } from stores/memoryStore
   - MigrationProgress from types/embedding
   - storage from storage/storage

2. Define constants:
   - MIGRATION_KEY = 'embedding_migration_complete'
   - BATCH_SIZE = 5 (process 5 memories then yield to UI)

3. Create singleton class `EmbeddingMigrationImpl`:
   - Private static instance
   - Private progress: MigrationProgress = { total: 0, completed: 0, status: 'idle' }
   - Private listeners: Set<(progress: MigrationProgress) => void>

4. Implement public methods:
   - `getInstance()`: Return singleton
   - `subscribe(listener)`: Subscribe to progress, return unsubscribe
   - `getProgress()`: Return current progress
   - `isMigrationNeeded()`: Return true if memories exist without embeddings AND migration not marked complete
   - `isComplete()`: Check MIGRATION_KEY in storage

5. Implement `startMigration(): Promise<void>`:
   - Guard: If not isMigrationNeeded(), return immediately
   - Guard: If status is 'running', return (prevent concurrent)
   - Guard: If EmbeddingService not ready, set error and return

   - Get all memories from memoryStore
   - Filter to memories without embeddings: filter((m) => !hasEmbedding(m.id))
   - Set progress: { total: filtered.length, completed: 0, status: 'running' }
   - Notify listeners

   - Process in batches:
     ```typescript
     for (let i = 0; i < memoriesToMigrate.length; i++) {
       const memory = memoriesToMigrate[i];

       try {
         const embedding = await EmbeddingService.embed(memory.content);
         storeEmbedding(memory.id, embedding);
       } catch (error) {
         console.error('[Migration] Failed to embed memory:', memory.id, error);
         // Continue with other memories
       }

       this.progress = { ...this.progress, completed: i + 1 };
       this.notifyListeners();

       // Yield to UI every BATCH_SIZE memories
       if ((i + 1) % BATCH_SIZE === 0) {
         await this.yieldToUI();
       }
     }
     ```

   - Mark complete:
     ```typescript
     storage.set(MIGRATION_KEY, 'true');
     this.progress = { ...this.progress, status: 'complete' };
     this.notifyListeners();
     ```

   - On error: Set status: 'error', set error message

6. Implement private `yieldToUI(): Promise<void>`:
   ```typescript
   return new Promise((resolve) => {
     if (typeof requestIdleCallback !== 'undefined') {
       requestIdleCallback(() => resolve(), { timeout: 1000 });
     } else {
       setTimeout(resolve, 0);
     }
   });
   ```

7. Implement `reset()`: Clear MIGRATION_KEY (for testing/re-migration)

8. Export singleton: `export const EmbeddingMigration = EmbeddingMigrationImpl.getInstance()`

Key behaviors:
- Idempotent: Skips memories that already have embeddings
- Resilient: Continues on individual failures
- Non-blocking: Yields to UI between batches
- Persistent: Marks complete in MMKV

3. Implement useEmbeddingMigration hook:
   ```typescript
   export function useEmbeddingMigration() {
     const [progress, setProgress] = useState<MigrationProgress>(
       EmbeddingMigration.getProgress()
     );
     const [isMigrationNeeded, setIsMigrationNeeded] = useState(false);

     useEffect(() => {
       // Subscribe to progress updates
       const unsubscribe = EmbeddingMigration.subscribe(setProgress);

       // Check if migration is needed
       setIsMigrationNeeded(EmbeddingMigration.isMigrationNeeded());

       return unsubscribe;
     }, []);

     const startMigration = useCallback(async () => {
       await EmbeddingMigration.startMigration();
     }, []);

     const percentComplete = progress.total > 0
       ? Math.round((progress.completed / progress.total) * 100)
       : 0;

     return {
       progress,
       startMigration,
       isMigrationNeeded,
       percentComplete,
     };
   }
   ```

4. Export the hook

Usage in app: Can show subtle migration progress or run silently in background.

   ```typescript
   /**
    * Initialize embedding system after LLM is ready
    * Call this from app initialization after chat model loads
    *
    * Flow:
    * 1. Check if embedding model is downloaded
    * 2. If yes, initialize EmbeddingService
    * 3. If service ready and migration needed, start migration
    */
   export async function initializeEmbeddingSystem(): Promise<void> {
     try {
       // Check if embedding model exists
       const isDownloaded = await EmbeddingService.isModelDownloaded();
       if (!isDownloaded) {
         console.log('[Embedding] Model not downloaded, skipping initialization');
         return;
       }

       // Initialize service
       await EmbeddingService.initialize();
       console.log('[Embedding] Service initialized');

       // Check and start migration if needed
       if (EmbeddingMigration.isMigrationNeeded()) {
         console.log('[Embedding] Starting migration for existing memories');
         // Don't await - run in background
         EmbeddingMigration.startMigration().catch((error) => {
           console.error('[Embedding] Migration failed:', error);
         });
       }
     } catch (error) {
       console.error('[Embedding] Initialization failed:', error);
       // Non-fatal - app works without embeddings
     }
   }
   ```

This provides a single entry point for the embedding system that can be called from the app's initialization flow (after LLM model loads).

Note: The actual trigger in app initialization will be added in Phase 6 or as part of polish, since it requires understanding the app's startup sequence. For now, the hook useEmbeddingModel can trigger this when embedding model download completes.
